The Foundation

The if statement

An if block runs code only when a condition is true. If false, the block is skipped entirely.

if condition then
  -- code that runs when condition is true
end
local enemyHealth = 0
if enemyHealth <= 0 then
  -- Award loot, play death animation, remove from world
end
Structure Rule

Every if must close with end. Forgetting this is one of the most common Lua syntax errors.

The Fallback

Adding else

The else block runs when the condition is false. One or the other always runs — never both, never neither.

local playerHealth = 45
if playerHealth <= 0 then
  -- Player is dead
else
  -- Player is still alive
end
playerHealth <= 0
→
TRUE → death logic
|
FALSE → alive logic

Multiple Conditions

Using elseif

Chain conditions with elseif. Lua checks them in order and stops at the first true one.

local health = 60
if health <= 0 then
  -- Dead
elseif health <= 25 then
  -- Critical
elseif health <= 50 then
  -- Hurt
else
  -- Healthy
end

With health = 60, only the else block runs. The first three conditions are all false.

A Common Pitfall

Order matters with elseif

Lua stops at the first true condition. More specific checks must come before broader ones.

✗ Wrong Order

if health < 100 then
  -- catches everything
elseif health < 25 then
  -- NEVER REACHED
end

✓ Correct Order

if health < 25 then
  -- Critical
elseif health < 100 then
  -- Damaged
else
  -- Full health
end

Going Deeper

Nested conditionals

You can place an if inside another if to check a secondary condition only after confirming the first.

local enemyLevel = 30
local isBoss     = true
if enemyLevel >= 25 then
  if isBoss then
    -- High-level AND a boss — raid warning
  else
    -- High-level but not a boss — elite behavior
  end
end
Keep It Readable

Avoid nesting more than two or three levels deep. Deeply nested code is hard to scan quickly.

Full Example

Zone entry gate check

function OnPlayerEnterZone()
  local playerLevel  = 18
  local hasQuestItem = true
  local zoneMinLevel = 20
  if playerLevel >= zoneMinLevel then
    -- Level requirement met
  elseif hasQuestItem then
    -- Under-leveled but has the pass
  else
    -- Block entry
  end
end

With playerLevel = 18 and hasQuestItem = true: first condition fails, second succeeds — entry allowed via quest item.

Cleaner Code

Boolean shortcuts

When your variable is already a boolean, skip == true. These are identical:

if isBoss == true then   -- verbose
if isBoss then             -- preferred

if isAlive == false then  -- verbose
if not isAlive then        -- preferred
