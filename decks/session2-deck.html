Modulo — What's it good for?

Modulo gives you the remainder after division. In game design: "every 5th kill triggers a bonus" — use killCount % 5 == 0

In Practice

Arithmetic in a game context

function OnAttack()
  local baseDamage    = 80
  local critMultiplier = 1.5
  local armor          = 20

  local critDamage  = baseDamage * critMultiplier
  local finalDamage = critDamage - armor
  -- finalDamage = (80 * 1.5) - 20 = 100
end

Reading complex expressions gets easier when you break them into steps. Use intermediate variables to make calculations self-documenting.

Asking Questions

Comparison operators

Comparisons always produce a boolean result — either true or false.

OperatorMeaningExampleResult
==Equal to5 == 5true
~=Not equal to5 ~= 3true
<Less than3 < 10true
>Greater than10 > 20false
<=Less than or equal5 <= 5true
>=Greater than or equal8 >= 10false
Common Mistake

Don't confuse == (comparison) with = (assignment). health == 0 asks "is health zero?" while health = 0 sets health to zero.

Joining Text

String concatenation

The .. operator joins strings together. This is how you build dynamic messages in your scripts.

local playerName = "Varek"
local zone        = "Thornwood"

local message = playerName .. " has entered " .. zone
-- "Varek has entered Thornwood"

local level   = 24
local display = "Level: " .. level
-- "Level: 24"

Order of Operations

Operator precedence

When multiple operators appear in one expression, Lua follows a specific order. Use parentheses to be explicit when in doubt.

PriorityOperators
1 (highest)^ (exponent)
2not, unary minus
3*, /, %
4+, -
5.. (concat)
6<, >, <=, >=, ==, ~=
7and
8 (lowest)or

When in doubt: use parentheses. (a + b) * c is always clearer.

Bringing It Together

A realistic damage formula

function CalculateDamage()
  local base       = 100
  local spellPower  = 40
  local resistance = 0.2
  local isCrit      = true

  local raw   = base + spellPower           -- 140
  local after = raw * (1 - resistance)       -- 112

  local isBigHit = isCrit and (after > 100) -- true
  local logMsg   = "Damage dealt: " .. after
end
