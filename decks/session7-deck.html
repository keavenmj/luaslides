The Core Concept

What is a table?

A table is Lua's one container type. Used as an array, it's an ordered list where each item has a numbered position.

-- A single variable holding one value
local enemyName = "Razorfang"

-- A table holding many values
local enemyNames = { "Razorfang", "Grimtusk", "Voidwalker" }
Lua is 1-indexed

Most languages start counting at 0. Lua starts at 1. The first item in a table is always at position 1.

Getting Values Out

Reading by index

Access a specific item using square brackets and its position number.

local spawnList = { "Razorfang", "Grimtusk", "Voidwalker" }

local first  = spawnList[1]   -- "Razorfang"
local second = spawnList[2]   -- "Grimtusk"
local third  = spawnList[3]   -- "Voidwalker"
[1]"Razorfang"
[2]"Grimtusk"
[3]"Voidwalker"

Reading an index that doesn't exist returns nil — not an error.

How Many Items?

The length operator: #

The # operator returns the number of items in a table.

local buffList = { "Strength", "Haste", "Regeneration" }
local count = #buffList   -- 3

Combine with a for loop to process every item:

for i = 1, #buffList do
  local currentBuff = buffList[i]
  -- Apply currentBuff to player
end

Growing the List

Adding items with table.insert

local inventory = { "Sword", "Shield" }

-- Append to the end
table.insert(inventory, "Potion")
-- { "Sword", "Shield", "Potion" }

-- Insert at position 2 (shifts everything right)
table.insert(inventory, 2, "Ring")
-- { "Sword", "Ring", "Shield", "Potion" }
Most Common Pattern

In game scripting you'll almost always append to the end — adding a kill to a log, pushing a buff onto a stack, queuing a spawn event.

Shrinking the List

Removing items with table.remove

table.remove removes an item by its position and returns the removed value.

local queue = { "Spawn A", "Spawn B", "Spawn C" }

-- Remove the first item (and get it back)
local next = table.remove(queue, 1)
-- next  = "Spawn A"
-- queue = { "Spawn B", "Spawn C" }

-- No position = removes the last item
table.remove(queue)
-- queue = { "Spawn B" }

Updating Values

Writing directly to an index

local waveEnemies = { "Grunt", "Grunt", "Grunt" }

-- Upgrade the third enemy
waveEnemies[3] = "Elite Grunt"
-- { "Grunt", "Grunt", "Elite Grunt" }

-- Add beyond the current end
waveEnemies[4] = "Boss"
-- { "Grunt", "Grunt", "Elite Grunt", "Boss" }
Watch Out

Writing to a non-sequential index (e.g. position 10 in a 3-item table) creates a "hole." The # operator behaves unpredictably with holes. Stick to sequential writes.

Processing Every Item

The ipairs loop

ipairs gives you both the index and the value on each iteration.

local lootTable = { "Gold Coin", "Iron Ore", "Health Potion" }

for i, item in ipairs(lootTable) do
  local msg = "Drop " .. i .. ": " .. item
end

-- If you only need the value, use _ for the index
for _, item in ipairs(lootTable) do
end

Putting It Together

A spawn queue with add and remove

function OnWaveSetup()
  local spawnQueue = { "Scout", "Scout", "Warrior" }

  -- Add a boss to the end
  table.insert(spawnQueue, "Warchief")

  local totalSpawned = 0
  for _, enemyType in ipairs(spawnQueue) do
    totalSpawned = totalSpawned + 1
  end
  -- totalSpawned = 4
end
