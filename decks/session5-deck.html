The Core Concept

What is a function?

A function is a named block of code that doesn't run until you call it. Think of it like a recipe card — writing it down doesn't cook the food.

-- DEFINING the function (nothing runs yet)
function ShowDamageText()
  -- logic goes here
end

-- CALLING the function (now it runs)
ShowDamageText()
Why They Matter

In a proprietary engine, your script is a function. The engine calls it when an event happens — on spawn, on hit, on death, on trigger. Understanding functions means understanding how your work gets executed.

Breaking It Down

Anatomy of a function

function ApplyDamage(target, amount)
  local finalDamage = amount * 0.9
  return finalDamage
end
  • function — keyword that starts the definition
  • ApplyDamage — the name you use to call it
  • target, amount — parameters: information passed in from outside
  • local finalDamage... — the body: logic that runs when called
  • return — sends a value back to the caller
  • end — closes the function

Passing Information In

Parameters

Parameters are placeholders in the definition. When you call the function, you fill them with real values (called arguments).

function GrantXP(playerName, xpAmount)
  local message = playerName .. " gained " .. xpAmount .. " XP"
end

-- Calling with real values
GrantXP("Varek", 250)
-- Inside: playerName = "Varek", xpAmount = 250

Getting Information Out

Return values

A function sends a result back with return. The caller can store that result in a variable.

function CalculateFinalDamage(base, multiplier, armor)
  local result = (base * multiplier) - armor
  return result
end

local damage = CalculateFinalDamage(100, 1.5, 20)
-- damage = 130
No Return is Fine Too

Not every function needs to return something. Functions that trigger effects, display text, or modify game state often just do the work.

Variables and Scope

What lives inside, stays inside

Variables declared with local inside a function only exist while that function is running.

function OnCombatStart()
  local combatTimer = 0  -- only exists inside
  combatTimer = combatTimer + 1
end
-- combatTimer does not exist here

Parameters work the same way — local to the function, created when it's called, gone when it ends.

Advanced Technique

Returning multiple values

Lua lets a function return more than one value at once.

function GetCombatResult(attackPower, defense)
  local damage = attackPower - defense
  local isCrit = (damage > 50)
  return damage, isCrit
end

local dmg, wasCrit = GetCombatResult(80, 20)
-- dmg = 60, wasCrit = true

Everything Together

A complete function example

function OnPlayerAttack(weaponDamage, targetArmor, isCritical)
  local base = weaponDamage
  if isCritical then
    base = base * 2
  end
  local final = base - targetArmor
  if final < 0 then final = 0 end
  return final
end

local dealt = OnPlayerAttack(50, 15, false)
-- dealt = 35
