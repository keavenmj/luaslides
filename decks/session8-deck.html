The Core Concept

What is a loop?

A loop runs the same block of code multiple times. Without loops, repetitive work requires writing the same code over and over:

-- Without loops: applying damage to 5 enemies manually
enemy1Health = enemy1Health - 20
enemy2Health = enemy2Health - 20
enemy3Health = enemy3Health - 20
-- ...doesn't scale.
Game Design Relevance

Loops power: spawning enemy waves, checking inventory, applying damage over time, running countdown timers, and much more.

Known Repetitions

The numeric for loop

Use a for loop when you know exactly how many times something should repeat.

for i = start, stop, step do
  -- code runs each iteration
end
function OnAoEAttack()
  local damage     = 30
  local enemyCount = 5
  for i = 1, enemyCount do
    -- Runs 5 times. i = 1, 2, 3, 4, 5
  end
end

The step argument is optional — if omitted, it defaults to 1.

Controlling the Step

Custom step values

-- Count up by 2: 0, 2, 4, 6, 8, 10
for i = 0, 10, 2 do end

-- Countdown: 10, 9, 8 ... 1
for seconds = 10, 1, -1 do
  -- Display seconds remaining
end
The Loop Variable

The variable i is automatically managed by the loop. You can read it inside the block, but don't manually change it — let Lua handle the incrementing.

Unknown Repetitions

The while loop

Use while when you don't know in advance how many times to repeat — only the stopping condition.

function OnChanneledSpell()
  local mana     = 100
  local manaCost = 15
  while mana >= manaCost do
    mana = mana - manaCost
    -- Trigger spell tick
  end
  -- Loop ends when mana drops below manaCost
end

Watch Out

The infinite loop trap

A while loop that never becomes false will run forever and freeze your engine.

✗ Infinite loop

local health = 100
while health > 0 do
  -- health never changes!
end

✓ Correctly terminating

local health = 100
while health > 0 do
  health = health - 10
end
Rule

Always ask: "what changes to make this condition false?" Every while loop must contain code that eventually makes that happen.

Run At Least Once

The repeat / until loop

Like a while loop, but the condition is checked at the end, so the block always runs at least once.

function OnLootRoll()
  local roll      = 0
  local threshold = 80
  repeat
    roll = roll + 1
    -- Check for drop...
  until roll >= threshold
end

Note: the condition is the exit condition — the loop runs until it becomes true.

Combining Concepts

Loops with conditionals inside

function OnWaveStart()
  local waveSize   = 8
  local spawnCount = 0
  for i = 1, waveSize do
    local slotIsEmpty = (i % 3 ~= 0)
    if slotIsEmpty then
      spawnCount = spawnCount + 1
    end
  end
  -- spawnCount = 6 (slots 3 and 6 skipped)
end
