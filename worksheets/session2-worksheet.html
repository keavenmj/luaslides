Write Your Own

Application

E1. Write an expression that evaluates to true only when a player's health is below 25 AND they are not already in a "low health" state (represented by a boolean variable isLowHealthWarningActive).

E2. A player has 80 base damage. Write the calculation for their damage after applying a 30% damage buff, then reducing it by a flat 15 armor value. Store the result in a variable called finalDamage.

E3. Build a display string that produces: "Grimtusk — Boss (Level 40)" using three separate variables for the name, type label, and level.

✓ Answer Key — Session 2
Section A — Arithmetic
A1: 22  |  A2: 52  |  A3: 85  |  A4: 2  |  A5: 15
A1: Multiplication runs before addition — 3 * 4 = 12, then 10 + 12 = 22.   A2: Parentheses first — 10 + 3 = 13, then 13 * 4 = 52.   A3: Division first — 30 / 2 = 15, then 100 - 15 = 85.   A4: 17 % 5 — 5 goes into 17 three times (15), remainder is 2.   A5: 2^4 = 16, then 16 - 1 = 15.
Section B — Comparisons
B1: true  |  B2: false  |  B3: true  |  B4: true  |  B5: false  |  B6: false
B6 is the common gotcha: 3 + 2 = 5, and 5 == 6 is false. Watch out for ~= — it's Lua's "not equal to," not != like other languages.
Section C — Logic
C1: true  |  C2: true  |  C3: true  |  C4: false  |  C5: false
C2: level >= 25 is false, but gold >= 100 is true — and or only needs one side true.   C3: not false = true.   C4: isDefeated is false and level < 5 is false — both false, so the whole or is false.   C5: gold >= 200 fails (gold is 150), so the and is false.
Section D — Concatenation
a: "Dawnseeker enters Ashfield"  |  b: "Level 12"  |  c: "Dawnseeker [Lv.12]"
Notice that Lua automatically converts numbers to strings when you use .. with them. The .. operator does not add spaces — you have to include any spaces explicitly in your string literals.
Section E1 — Sample Answer
local health = 20
local isLowHealthWarningActive = false
local shouldWarn = (health < 25) and (not isLowHealthWarningActive)
Using not isLowHealthWarningActive avoids triggering the warning if it's already showing. Both conditions must be true for shouldWarn to be true.
Section E2 — Sample Answer
local baseDamage = 80
local buffed = baseDamage * 1.30     -- 104
local finalDamage = buffed - 15      -- 89
A 30% buff means multiplying by 1.30 (the original 100% plus the 30% bonus). Breaking into two lines makes each step easy to read and debug.
Section E3 — Sample Answer
local name  = "Grimtusk"
local label = "Boss"
local level = 40
local display = name .. " — " .. label .. " (Level " .. level .. ")"
Notice the spacing: the em dash and spaces around it are literal characters inside the string. Every character in the final output must appear somewhere in the expression.
