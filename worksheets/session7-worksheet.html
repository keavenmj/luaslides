Predict the Result

Loop trace

What are the values of total and highCount after this function runs?

function OnScoreTally()
  local scores    = { 12, 45, 8, 67, 30 }
  local total     = 0
  local highCount = 0

  for _, score in ipairs(scores) do
    total = total + score
    if score >= 30 then
      highCount = highCount + 1
    end
  end
end

Write Your Own

Application

E1. Write a function called OnBuffApply that receives a table called activeBuffs and a string newBuff. If the table has fewer than 5 items, add the new buff and return true. Otherwise return false (buff limit reached).

E2. Write a function called OnWaveSummary that receives a table of enemy names called defeatedEnemies. Loop through it with ipairs and build a single string listing all names separated by ", ". Return that string. Example output for {"Scout","Grunt","Warchief"}: "Scout, Grunt, Warchief"

✓ Answer Key — Session 7
A — Index Reads
a = "Fireball" | b = "Blink" | c = nil | d = 4
c: Index 5 doesn't exist in a 4-item table, so it returns nil — not an error. d: The # operator counts 4 items.
B — Mutation Trace
After insert "Mage": {"Ranger","Healer","Tank","Mage"}
After insert "Scout" at 1: {"Scout","Ranger","Healer","Tank","Mage"}
After remove at 3: {"Scout","Ranger","Tank","Mage"}
After party[2]="Assassin": {"Scout","Assassin","Tank","Mage"}
Positional insert shifts everything at and after the target index one step right. Positional remove (index 3) removes "Healer" — the item currently sitting at index 3 after the two inserts, not the original item at index 3.
C1 — Off-by-One
Problem: #rewards - 1 = 2, so the loop stops at index 2 and never reads index 3.
Fixed: for i = 1, #rewards do. The - 1 is unnecessary and causes the last item to be skipped. This is one of the most common loop errors with tables.
C2 — Wrong Remove
table.remove without a position removes the last item — "Buckler", not "Cursed Blade".
Correct call: table.remove(inventory, 2). "Cursed Blade" is at index 2, so you must specify that position explicitly.
D — Loop Trace
total = 162 | highCount = 3
12 + 45 + 8 + 67 + 30 = 162. Scores >= 30: 45, 67, and 30 qualify — so highCount = 3. Score 12 and 8 don't qualify.
E1 — Sample Answer
function OnBuffApply(activeBuffs, newBuff)
  if #activeBuffs < 5 then
    table.insert(activeBuffs, newBuff)
    return true
  else
    return false
  end
end
Using #activeBuffs to check the count before inserting keeps the logic clean and avoids the need to track size separately.
E2 — Sample Answer
function OnWaveSummary(defeatedEnemies)
  local result = ""

  for i, name in ipairs(defeatedEnemies) do
    if i == 1 then
      result = name
    else
      result = result .. ", " .. name
    end
  end

  return result
end
The first item is handled separately to avoid a leading ", ". Every subsequent item gets prepended with the separator. For an empty table, result stays "" and that's returned — which is reasonable default behavior.
