function OnEnemyDefeated(enemy)
  local name       = enemy.name
  local xp         = enemy.xpReward
  local isBoss     = enemy.isBoss
  local bonusXP    = 0
  local logMessage = ""

  if isBoss then
    bonusXP    = xp * 2
    logMessage = "Boss defeated: " .. name
  else
    bonusXP    = xp
    logMessage = "Enemy defeated: " .. name
  end
end

✓ Answer Key — Session 6
A — Property Reads
pName = "Lyria" | pLevel = 22 | pHealthPct = 40 | pGuild = nil
pHealthPct: 80 / 200 * 100 = 40. The calculation chains directly on the property reads — no need to store them in separate variables first. pGuild: The schema says guildName may be nil; with our test data it is nil. Your code shouldn't crash from reading nil, but it will crash if you try to concatenate it into a string without checking first.
B — Enemy Defeated Trace
name = "Vaultkeeper" | xp = 400 | isBoss = true | bonusXP = 800 | logMessage = "Boss defeated: Vaultkeeper"
Because isBoss is true, the first branch runs. bonusXP = 400 * 2 = 800. The log message is built by concatenating the literal string with the name variable. The else branch never runs.
C1 — Nil Concatenation
Problem: if player.guildName is nil, concatenating it with ".." crashes.
function OnGuildCheck(player)
  local guildName = player.guildName
  local display

  if guildName == nil then
    display = "No guild"
  else
    display = "Guild: " .. guildName
  end
end
Always check for nil before using an optional property in string operations or arithmetic.
C2 — Swapped Objects
playerHP reads from enemy.health, and enemyHP reads from player.health — they're backwards.
function OnEncounter(player, enemy)
  local playerHP = player.health   -- correct
  local enemyHP  = enemy.health    -- correct
  if playerHP > enemyHP then
    -- "Player has more health"
  end
end
This is a very easy mistake to make with multiple objects. The fix is to always read carefully: which object does this property belong to?
D1 — Sample Answer
function OnZoneEnter(player, zone)
  local pLevel   = player.level
  local zMinLvl  = zone.minimumLevel
  local zName    = zone.name

  if pLevel >= zMinLvl then
    return true, "Welcome to " .. zName
  else
    local needed = zMinLvl - pLevel
    return false, "Need " .. needed .. " more levels to enter " .. zName
  end
end
Reading properties into local variables first keeps the logic clean. The function returns two values — a boolean and a string — so the caller can act on both.
D2 — Sample Answer
function OnWaveComplete(zone)
  local count      = zone.spawnCount
  local isHostile  = zone.isHostile
  local baseXP     = 50
  local totalXP    = 0

  for i = 1, count do
    if isHostile and (i % 2 == 0) then
      totalXP = totalXP + (baseXP * 2)
    else
      totalXP = totalXP + baseXP
    end
  end

  return totalXP
end
This example combines every major concept from the course: external data reads, a for loop, a conditional inside the loop, arithmetic, and a return value. If spawnCount = 6 and isHostile = true: enemies 1,3,5 give 50 XP each (150 total), enemies 2,4,6 give 100 XP each (300 total). Total = 450.
