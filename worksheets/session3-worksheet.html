Write Your Own

Application

D1. Write a conditional block that checks an NPC's relationship value (a number from -100 to 100) and sets a string variable called attitude to one of three values: "Hostile" (below 0), "Neutral" (0 to 49), or "Friendly" (50 and above).

D2. A spell can only be cast when: the player has enough mana AND the spell is not on cooldown. Write this as a single if condition and a comment describing what happens in each branch.

✓ Answer Key — Session 3
A1
Branch B runs
gold = 80, so gold >= 100 is false. The else block is the fallback and always runs when the if condition fails.
A2
Branch B runs
health = 20. Branch A requires health <= 0 — false. Branch B requires health <= 25 — true. Lua stops here and runs Branch B. Branch C and D are never checked.
A3
Branch B runs
Branch A requires both level >= 30 AND hasKey. Level passes but hasKey is false, so the and fails. Branch B only requires level >= 30 — that's true. Branch B runs.
B1
Missing "end" to close the if block
Every if in Lua must be closed with end. Without it, Lua will throw a syntax error because it doesn't know where the block finishes.
B2 — Problem
The first condition (damage > 0) catches everything — "Heavy damage" and "Massive damage" branches are unreachable.
Any damage above 50 or 100 also satisfies damage > 0, so Lua stops there and never evaluates the later branches.
B2 — Fixed Version
if damage > 100 then
  -- "Massive damage"
elseif damage > 50 then
  -- "Heavy damage"
elseif damage > 0 then
  -- "Some damage"
end
Specific (most restrictive) conditions first, general last.
C — Fill in the Blank
C1: playerLevel == 15  |  C2: gold >= 100 and gold <= 300  |  C3: not isInGuild  |  C4: playerLevel >= 10 or gold >= 500
C2: You need two conditions joined with and to form a range. Neither > 100 alone nor < 300 alone captures the "between" requirement.   C3: not isInGuild is the cleanest way to check for a false boolean — no need for isInGuild == false.
D1 — Sample Answer
local relationship = 65
local attitude

if relationship < 0 then
  attitude = "Hostile"
elseif relationship < 50 then
  attitude = "Neutral"
else
  attitude = "Friendly"
end
Declaring attitude before the block and then assigning inside each branch is valid and clean. The final else handles 50 and above without needing an explicit >= 50 check.
D2 — Sample Answer
local mana = 80
local manaCost = 60
local isOnCooldown = false

if mana >= manaCost and not isOnCooldown then
  -- Cast the spell
else
  -- Cannot cast: show error message
end
Using not isOnCooldown is cleaner than isOnCooldown == false. The and requires both conditions to be satisfied simultaneously.
