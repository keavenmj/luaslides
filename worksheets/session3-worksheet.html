✓ Answer Key — Session 1

Review your answers against the explanations below. Focus on understanding why each answer is correct, not just whether you got it right.

Section A — Data Types
string, number, boolean, nil, number, boolean, string, number
Key things to notice: goldAmount = 0 is still a number even though it's zero. The value true without quotes is a boolean. If it had quotes ("true"), it would be a string — which is a common mistake.
Section B1 — Variable Name with a Space
local playerLevel = 10
Variable names cannot contain spaces. Lua would read player level as two separate tokens and throw a syntax error. Use camelCase: playerLevel.
Section B2 — Boolean in Quotes
local hasMount = true
Wrapping true in quotes makes it the string "true", not the boolean true. These behave completely differently in logic checks. Always write boolean values without quotes.
Section B3 — Re-declaring with local
armorRating = 145
You only use local when declaring a variable for the first time. Using it again creates a new, separate variable that shadows the original. To update the value, just write the name and assign the new value — no local keyword.
Section C — Trace the Values
gold = 40  |  bonusGold = 0  |  totalGold = 75
Step by step: totalGold is set to 50 + 25 = 75 first. Then gold is updated to 50 - 10 = 40. Finally bonusGold is set to 0. Notice that changing gold and bonusGold after the fact does NOT change totalGold — it was already calculated.

Section D2 — Health Tracking
local health = 100
health = health - 35    -- takes damage: now 65
health = health + 20    -- heals: now 85
The key here is updating the same variable using its own current value. Declaring a new variable each time would be incorrect — the problem specifically asks for one variable.
Section D3 — Why Use Local?
Sample answer:
Local variables only exist inside the function where they're created. This means they can't accidentally interfere with other scripts or functions that might use the same name. Global variables linger and can be changed from anywhere, which makes bugs much harder to track down — especially in a game engine where many scripts may be running at once.
